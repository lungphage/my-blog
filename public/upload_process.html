步骤详解

1
复制 HTML 文件到 public/

源文件路径：I:\本机AI_agent结果合集文件夹\claude_2\funannotate_final_guide.html

目标路径：C:\Users\11296\Documents\my-blog\public\funannotate_final_guide.html

Copy-Item "I:\本机AI_agent结果合集文件夹\claude_2\funannotate_final_guide.html" `
  "C:\Users\11296\Documents\my-blog\public\" -Force
为什么放 public/？静态 HTML 文件直接放入 public/ 目录，Astro 不会处理其中的 ${...} 模板表达式。 如果放在 src/pages/ 中，bash 脚本里的变量会被 Astro 当作模板语法报错。
2
添加返回按钮

在 HTML 文件的 <body> 标签后、原有内容前，插入导航栏：

<div style="background:#fff;padding:0.8em 1.5em;border-bottom:1px solid #e9d8fd;
  display:flex;align-items:center;gap:1em;position:sticky;top:0;z-index:100;">
  <a href="/blog/" style="color:#553c9a;text-decoration:none;">← 返回博客</a>
  <span style="color:#718096;font-size:0.85em;">流光镜影</span>
</div>

效果：页面顶部出现固定的紫色导航栏，点击可返回博客列表。

3
编辑博客列表页

编辑 src/pages/blog/index.astro，在 articles 数组中添加新条目：

{
  title: 'Funannotate Singularity 完整操作记录 — Fungus_sp_MAG1',
  desc: 'funannotate 完整操作流程记录，从 clean 到 annotate 全步骤实战',
  date: '2026年5月21日',
  url: '/funannotate_final_guide.html',
  icon: '📋',
  category: 'software',
},

字段说明：

字段说明
title文章标题，显示在卡片中
desc摘要描述，显示在标题下方
date发布日期
url链接地址，指向 public/ 中的 HTML 文件
icon卡片图标（emoji）
category分类：software / paper / thoughts
4
构建验证

运行构建命令确认无报错：

cd C:\Users\11296\Documents\my-blog
npm run build

预期输出：✓ Completed，无 ERROR。

5
提交并推送
git add .
git commit -m "add: Funannotate Singularity 完整操作记录"
git push
推送后 Cloudflare Pages 自动部署，约 1-2 分钟后网站更新。

关键规范（来自 AGENTS.md）

  • 静态 HTML 文件必须放 public/，不能放 src/pages/（避免 ${...} 模板冲突）
  • 每篇文章必须在 src/pages/blog/index.astro 的 articles 数组中手动添加条目
  • 每篇文章必须有返回按钮（← 返回博客）
  • 分类标签：software（软件安装）、paper（文献阅读）、thoughts（想法）
  • npm run build 是唯一的验证手段（无 lint / typecheck）
  • Git push 到 main 触发 Cloudflare Pages 自动部署
