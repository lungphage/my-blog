访客追踪系统部署指南

系统架构

组件文件说明
Cloudflare Workercloudflare-worker/worker.js访客记录 API，运行在 Cloudflare 边缘节点
前端追踪脚本public/visitor-tracker.js每个页面加载时自动上报访客信息
管理后台public/visitors.html查看访客记录（密码保护）

记录的信息

字段来源说明
ip请求头 CF-Connecting-IP访客真实 IP 地址
country请求头 CF-IPCountry访客国家（Cloudflare 自动识别）
browserUser-Agent 解析浏览器类型和版本
osUser-Agent 解析操作系统
deviceUser-Agent 解析设备类型（PC/手机/平板）
pageJavaScript访问的页面路径
timeWorker访问时间（ISO 8601）
referrerJavaScript来源页面

部署步骤

1
创建 KV 命名空间

在 Cloudflare Dashboard 中：

  1. 登录 Cloudflare Dashboard
  2. 左侧菜单 → Workers & Pages → KV
  3. 点击 Create a namespace
  4. 名称填 visitor-logs，点击 Add
  5. 复制生成的 ID
2
创建 Worker
  1. 在 Workers & Pages 页面点击 Create application → Create Worker
  2. 名称填 blog-visitor-tracker
  3. 点击 Deploy（先用默认代码）
  4. 进入 Worker 设置 → Variables → KV Namespace Bindings
  5. 添加绑定：Variable name = VISITOR_KV，KV Namespace = 选刚创建的 visitor-logs
  6. 回到 Settings → Triggers → 复制 Worker 的 URL（类似 https://blog-visitor-tracker.xxx.workers.dev）
3
部署 Worker 代码

在 Worker 编辑页面，将默认代码替换为 cloudflare-worker/worker.js 的内容。

修改第 4 行的密码：

const ADMIN_PASSWORD = "your_password_here"; // 改成你的密码

点击 Save and deploy。

4
修改前端配置

修改 public/visitor-tracker.js，将 Worker URL 改成你的：

const WORKER_URL = "https://blog-visitor-tracker.xxx.workers.dev";

修改 public/visitors.html，同样更新 Worker URL：

const WORKER_URL = "https://blog-visitor-tracker.xxx.workers.dev";
5
提交并部署

与评论的关联

如何知道谁评论了？
Giscus 评论会显示评论者的 GitHub 用户名。你可以在管理后台 /visitors.html 查看同一时间段的访客 IP 和设备信息，通过时间戳和页面路径与 GitHub Discussions 中的评论进行匹配。

GitHub 通知：有人评论时，你会收到 GitHub 通知（站内 + 邮件），可以立即回复。
