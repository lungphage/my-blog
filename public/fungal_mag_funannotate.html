运行环境与基础命令模板

本流程所有 funannotate 命令均使用以下 Singularity 模板

项目路径/说明
funannotate 容器~/soft/funannotate_latest.sif
funannotate 可执行文件容器内 /venv/bin/funannotate
GeneMark-ES/ET 目录/home/liuzifeng/soft/genemark/gmes_linux_64_4
GeneMark 许可证~/.gm_key
Perl 模块目录~/perl5/lib/perl5/
▶ 简写命令（clean / sort / mask / train / update / annotate / compare）
singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate <子命令> [参数...]
▶ predict 专用命令（需要 GeneMark PATH，仅此步骤使用）
singularity exec ~/soft/funannotate_latest.sif \
    bash -c 'export PATH=/home/liuzifeng/soft/genemark/gmes_linux_64_4:$PATH \
             && /venv/bin/funannotate "$@"' \
    -- predict [参数...]
为什么 predict 要用长命令：GeneMark-ES/ET（gmes_petap.pl）只在 predict 阶段被调用，其他步骤均不涉及 GeneMark，直接用简写即可。
bash -c '...' -- "$@" 将 GeneMark 目录前置到 PATH 同时保留容器原有 /venv/bin，不用 --env 是因为它会覆盖整个容器 PATH。
0

EukCC — 真菌 MAG 质量评估 建议先做

评估真核 MAG 的完整性与污染率，过滤低质量 bin

eukcc folder \
    --out eukcc_out/ \
    --threads 16 \
    bins/

过滤标准（MIMAG）：

等级完整性污染率建议
高质量 HQ≥ 90%≤ 5%可直接注释
中质量 MQ≥ 50%≤ 10%可注释，结果不完整
低质量 LQ< 50%—一般丢弃
也可用 BUSCO 粗估完整性：
busco -i mag.fasta -l fungi_odb10 -o busco_qc -m genome --cpu 16
1

funannotate clean — 清理输入序列 必须

去除短 contig、对齐重复序列，规范化 fasta 格式

singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate clean \
    -i mag.fasta \
    -o mag_clean.fasta \
    --minlen 1000
输入mag.fasta — 原始 MAG
输出mag_clean.fasta — 清理后序列
参数说明默认值
--minlen保留 contig 最小长度（bp）500
--pident去除重复序列的相似度阈值 %95
--exhaustive使用更严格的去重（较慢）—
2

funannotate sort — 排序与重命名 必须

按长度降序排列 contig，统一命名格式（避免 Augustus 不识别的特殊字符）

singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate sort \
    -i mag_clean.fasta \
    -o mag_sorted.fasta \
    -b scaffold
输入mag_clean.fasta
输出mag_sorted.fasta — contig 重命名为 scaffold_1, scaffold_2…
参数说明
-b scaffoldcontig 名称前缀，也可用 contig、物种缩写等
3

funannotate mask — 重复序列屏蔽 必须

对转座子、重复序列进行软屏蔽（lowercase），提高后续基因预测准确性

singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate mask \
    -i mag_sorted.fasta \
    -o mag_masked.fasta \
    --cpus 16
输入mag_sorted.fasta
输出mag_masked.fasta — 重复区域转为小写字母
内部调用 RepeatMasker（自带数据库）+ RepeatModeler（从头建库）。
如已有 .out 文件可用 --repeatmasker_out 跳过重新计算。
软屏蔽 vs 硬屏蔽：funannotate 使用软屏蔽（小写字母），不要用 -N 硬屏蔽（N 字符）替换，会导致基因预测缺失边界信息。
4

funannotate train — 训练 Augustus 模型 可选，有 RNA-seq 时做

用 RNA-seq 数据运行 Trinity 组装转录本，训练物种特异性 Augustus 基因模型

singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate train \
    -i mag_masked.fasta \
    -o fun_out \
    --left   RNA_R1.fastq.gz \
    --right  RNA_R2.fastq.gz \
    --stranded RF \
    --jaccard_clip \
    --species "Fungus_sp_MAG1" \
    --cpus 16
输入mag_masked.fasta + RNA-seq reads
输出fun_out/training/ — Trinity 组装、PASA 比对结果、Augustus 训练数据
参数说明
--stranded RF链特异性文库方向（常见：RF 或 FR，不确定则省略）
--jaccard_clip真菌基因密集，推荐开启，减少嵌合转录本
--single单端 reads 时替代 --left/--right
--trinity提供已有 Trinity 组装结果跳过重新组装
无 RNA-seq 时：直接跳到 Step 5（predict），funannotate 会使用已有的 Augustus 物种模型或 BUSCO 内置模型。
5

funannotate predict — 基因预测 核心步骤

整合 Augustus、GeneMark-ES/ET、GlimmerHMM、SNAP、BUSCO 等多种工具进行 ab initio 预测，结合转录本证据生成最终基因模型

singularity exec ~/soft/funannotate_latest.sif \
    bash -c 'export PATH=/home/liuzifeng/soft/genemark/gmes_linux_64_4:$PATH \
             && /venv/bin/funannotate "$@"' \
    -- predict \
    -i mag_masked.fasta \
    -o fun_out \
    --species "Fungus_sp_MAG1" \
    --busco_seed_species botrytis_cinerea \
    --busco_db fungi_odb10 \
    --organism other \
    --cpus 16
输入 mag_masked.fasta
fun_out/training/（若有 train 结果）
输出 fun_out/predict_results/
.gff3 基因模型
.proteins.fa 蛋白质序列
.transcripts.fa 转录本序列
.scaffolds.fa 基因组序列
参数说明
--species物种名（用于 Augustus 模型命名），建议用下划线代替空格
--busco_seed_species初始 Augustus 训练的种子物种，选择亲缘最近的已知模型
--busco_dbBUSCO 数据库，真菌通常用 fungi_odb10（见末尾数据库列表）
--organism other非脊椎动物填 other（真菌、植物等均用此选项）
--augustus_species直接指定已有 Augustus 物种模型（跳过重新训练）
--genemark_mode ET指定 GeneMark 模式：ET（有 RNA-seq）或 ES（无）
--min_intronlen最小内含子长度，真菌默认 10 bp
--max_intronlen最大内含子长度，真菌默认 3000 bp
常用真菌 Augustus 种子模型（--busco_seed_species）：
子囊菌：botrytis_cinerea / aspergillus_fumigatus / fusarium_graminearum
担子菌：cryptococcus_neoformans_neoformans_B / coprinus_cinereus
壶菌：rhizopus_oryzae
6

funannotate update — 更新基因模型 可选，有 RNA-seq 时做

用 PASA 将转录本证据整合到基因模型中，修正 UTR、可变剪接等

singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate update \
    -i fun_out \
    --cpus 16
输入fun_out/ 目录（含 predict_results 和 training）
输出fun_out/update_results/ — 更新后的 gff3/蛋白质序列
update 完成后，annotate 步骤会自动优先使用 update_results/ 中的模型。
7

funannotate annotate — 功能注释 核心步骤

对预测基因进行 InterProScan、Swiss-Prot BLAST、GO、KEGG、CAZyme、MEROPS 等功能数据库注释

singularity exec ~/soft/funannotate_latest.sif \
    /venv/bin/funannotate annotate \
    -i fun_out \
    --busco_db fungi_odb10 \
    --cpus 16
输入fun_out/ 目录（含 predict 或 update 结果）
输出fun_out/annotate_results/
.gbk GenBank 格式
.gff3 基因组注释
annotation.txt 功能注释汇总
interproscan.xml InterPro 结果
pfam.txt, merops.txt, cazy.txt
参数说明
--sbtNCBI submission 模板 .sbt 文件（提交 GenBank 时需要）
--eggnog提供 eggNOG-mapper 预先运行的 .emapper.annotations 文件
--iprscan提供 InterProScan 预先运行的 xml 文件（节省时间）
--busco_dbBUSCO 数据库（与 predict 步骤保持一致）
--fix提供人工修正的 GFF3 文件
InterProScan 在容器内运行，首次运行会下载数据库（约 20 GB），耗时较长。
建议提前在容器外运行 InterProScan 后用 --iprscan 传入结果文件。

输出compare_out/

基因家族扩张/收缩（CAFE）

多个真菌 MAG 批量处理示例

循环处理多个 MAG

#!/bin/bash
# 批量注释目录 mags/ 下的所有 fasta 文件
SIF=~/soft/funannotate_latest.sif
GMPATH=/home/liuzifeng/soft/genemark/gmes_linux_64_4

for f in mags/*.fasta; do
    NAME=$(basename "$f" .fasta)
    echo "=== 处理 $NAME ==="

    # clean（简写）
    singularity exec $SIF /venv/bin/funannotate clean \
        -i $f -o ${NAME}_clean.fasta --minlen 1000

    # sort（简写）
    singularity exec $SIF /venv/bin/funannotate sort \
        -i ${NAME}_clean.fasta -o ${NAME}_sorted.fasta -b scaffold

    # mask（简写）
    singularity exec $SIF /venv/bin/funannotate mask \
        -i ${NAME}_sorted.fasta -o ${NAME}_masked.fasta --cpus 16

    # predict（需要 GeneMark PATH，用长命令）
    singularity exec $SIF \
        bash -c 'export PATH=${GMPATH}:$PATH && /venv/bin/funannotate "$@"' \
        -- predict \
        -i ${NAME}_masked.fasta \
        -o fun_${NAME} \
        --species ${NAME} \
        --busco_seed_species botrytis_cinerea \
        --busco_db fungi_odb10 \
        --organism other \
        --cpus 16

    # annotate（简写）
    singularity exec $SIF /venv/bin/funannotate annotate \
        -i fun_${NAME} --busco_db fungi_odb10 --cpus 16

    echo "=== $NAME 完成 ==="
done
批量脚本在 Linux 服务器上创建，避免从 Windows 粘贴（CRLF 换行符会导致 syntax error）。
已有脚本文件时先检查：cat -A script.sh | head -5，行尾出现 ^M 则运行 sed -i 's/\r//' script.sh。
