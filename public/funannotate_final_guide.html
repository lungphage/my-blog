问题一：exonerate 符号缺失 非致命

报错信息

exonerate: symbol lookup error: exonerate: undefined symbol: g_string_free_and_steal
     Progress: 0 complete, 264277 failed, 0 remaining
[...]: Exonerate finished in 0:05:44: found 0 alignments

根本原因

容器内 exonerate 二进制链接了 GLib ≥ 2.76（该版本引入 g_string_free_and_steal），但宿主机运行时加载的 GLib 版本不包含此符号，导致动态链接失败。

影响评估

影响范围蛋白同源证据（protein homology weight）缺失，264,277 个比对任务全部失败
是否中断流程否。funannotate 以 0 alignments 继续，GeneMark / Augustus / SNAP / GlimmerHMM 正常运行
对结果影响轻微，ab initio 预测器仍可产出基因模型
处理方式：忽略，流程不中断。 如需彻底修复，需在容器内替换为与宿主 GLib 兼容的 exonerate 版本，或升级宿主机 GLib。本次流程选择忽略此报错继续运行。

根本原因分析

通过读取 augustus_parallel.py 源码（第 89 行）定位问题：

# augustus_parallel.py 第 39、46~64、89 行关键逻辑：

funannotate BUSCO 步骤在 fun_1/predict_misc/ab_initio_parameters/augustus 创建了一个不完整的 augustus config 副本（仅含物种参数，缺少全局配置文件）。Augustus 以此为 config 路径启动时因找不到必要文件而立即崩溃，输出文件从未生成。

最终解决方案：自定义 Augustus 并行脚本 成功 ✓

完全绕过 augustus_parallel.py，使用自定义 Python 脚本直接调用 subprocess + multiprocessing.Pool 并行运行 augustus，再将结果通过 --augustus_gff 传入 funannotate。

拆分 genome → scaffold FASTA
→
Pool(30) 并行运行 augustus
→
合并为 augustus.gff3
→
--augustus_gff 传入 funannotate
run_augustus_parallel.py
#!/usr/bin/env python3
import subprocess, os
from multiprocessing import Pool
from Bio import SeqIO

# ── 配置项 ──────────────────────────────────────────
GENOME  = "fun_1/predict_misc/genome.softmasked.fa"
CFG     = "/home/liuzifeng/2test_funannotate/fun_1/predict_misc/ab_initio_parameters/augustus"
OUTDIR  = "/home/liuzifeng/2test_funannotate/aug_tmp"
CPUS    = 30
SPECIES = "fungus_sp_mag1"
OUT_GFF = "fun_1/predict_misc/augustus.gff3"
# ────────────────────────────────────────────────────

os.makedirs(OUTDIR, exist_ok=True)

def run_aug(rec_id):
    fa  = f"{OUTDIR}/{rec_id}.fa"
    gff = f"{OUTDIR}/{rec_id}.gff3"
    r = subprocess.run(
        ["augustus", f"--species={SPECIES}",
         f"--AUGUSTUS_CONFIG_PATH={CFG}",
         "--softmasking=1", "--gff3=on", fa],
        capture_output=True, text=True
    )
    with open(gff, 'w') as f_out:
        f_out.write(r.stdout)
    return rec_id

# 拆分基因组为单 scaffold FASTA
rec_ids = []
with open(GENOME) as fh:
    for rec in SeqIO.parse(fh, "fasta"):
        with open(f"{OUTDIR}/{rec.id}.fa", 'w') as out:
            SeqIO.write(rec, out, "fasta")
        rec_ids.append(rec.id)
print(f"Split {len(rec_ids)} scaffolds, running with {CPUS} CPUs...")

# 并行运行 augustus
with Pool(CPUS) as pool:
    for i, rid in enumerate(pool.imap_unordered(run_aug, rec_ids)):
        if (i + 1) % 500 == 0:
            print(f"  {i+1}/{len(rec_ids)} done")

# 合并所有 GFF3
with open(OUT_GFF, 'w') as out:
    for rid in rec_ids:
        gff = f"{OUTDIR}/{rid}.gff3"
        if os.path.exists(gff) and os.path.getsize(gff) > 0:
            with open(gff) as f_in:
                out.write(f_in.read())

total = sum(1 for _ in open(OUT_GFF))
print(f"Done: {OUT_GFF}  ({total} lines)")

关键经验总结

问题根本原因解决方案
Augustus 预测全部失败 Singularity 容器内 /usr/share/augustus/config 只读，训练参数无法写入 cp -r 提取到宿主机，--bind ~/augustus_config:/usr/share/augustus/config 挂载

每次重跑都重训 Augustus funannotate predict 没有缓存 Augustus 训练步骤 --augustus_gff 传入预计算 GFF3，完全跳过 Augustus 预测步骤
